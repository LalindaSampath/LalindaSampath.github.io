Is node.js a replacement for php

2016-10-13
Software Engineering
NodeJS, PHP, UI Development

Few years ago partnership between javascript and php was simple. Javascript handled most of the client side tasks and php handled all the server-side tasks. Then Ryan Dahl and other developers working at Joyent discovered they could get JavaScript running on the server-side, which is a open-source runtime system built on Chrome’s V8 JavaScript engine. Node.js is popular for building real-time web APIs.

###Where PHP wins:
– The most popular platforms for building web sites such as wordpress, Joomla and drupal are written in PHP, and those are open-source.
– Many templates available in web are written in PHP. You only need to modify and use it to cater our needs.
– Simplicity and easy to code. No compilers, no deployments, no JAR files or preprocessors, just your code editor and some php files.
– PHP was built to exist with MySQL. If MySQL isn’t your preference you can choose any DBMS technologies from ORACLE to Microsoft.

###Where Node.js wins:
From all the homework I’ve been doing in the last few days, I figured out the following information.
– very good when you need to do several things at the same time (Multitasking)
– event-based, so all the wonderful Ajax-like stuff can be done on the server side
– lets us share code between the browser and the backend
– Especially suited for real-time applications. Using a technique known as “long-polling”, you can write an application that sends updates to the user in real time. This means you can create a browser-based chat application in Node.js that takes almost no system resources to serve a great many clients.
It’s worth mentioning that Ruby and Python both have tools to do this sort of thing , but that Node.js does it exceptionally well.When you use something like Node.js, the server has no need of maintaining separate threads for each open connection.
– NPM: The Node Package Manager
NPM is very similar to Ruby gems, a set of publicly available, reusable components, available through easy installation via an online repository. All the details of package modules can be found on the NPM website, https://npmjs.org/
There are tons of really useful packages out there, available to all
– Popularity of MEAN Stack
The acronym MEAN stands for ‘MongoDB ExpressJS AngularJS NodeJS’.
The major benefit of the MEAN stack is that it’s extremely quick to prototype with. Node.js allows you to use Javascript on the backend as well as the frontend which can save you from having to learn a separate language. In addition, the NoSQL nature of MongoDB allows you to quickly change and alter the data layer without having to worry about migrations.

###Where Node.JS should be used?
– Chat
Chat is the most real-time, multi-user application.
– API on top of an Object DB
Can be used to extract data from Object DBs such as MongoDB.
– Queued Inputs
When the server receiving high amount of concurrent data, your DB might be ran into problems. But Node.js can easily handle the concurrent connections themselves.
– Data Streaming
– PROXY
Node.js is easily employed as a server-side proxy where it can handle a large amount of simultaneous connections in a non-blocking manner

###Where Node.JS should not be used?
– SERVER-SIDE WEB APPLICATION W/ A RELATIONAL DB BEHIND
Relational DB tools for Node.js are still in their early stages, not mature yet.
– HEAVY SERVER-SIDE COMPUTATION/PROCESSING

###Conclusion
Of course, the ending of Node.JS isn’t written yet, as well as PHP.
Remember that Node.js was never created to solve the compute scaling problem. It was created to solve the I/O scaling problem, which it does really well.
For every coder talking about the purity of Node.js and the simplicity of JavaScript everywhere, there’s another who’s happy with the deep code base and long-understood stability of PHP.
